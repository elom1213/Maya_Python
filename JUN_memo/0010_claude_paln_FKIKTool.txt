A00120_FKIK — 레거시 FKIK Tool 모듈화 / 리팩토링 + Qt UI 이식

 Context (왜 만드는가)

 01_Modules/JUN_PY_FKIK_Tool_V02_01.py(964줄, 단일 파일)는 현재도 사용 중인 FK/IK 매칭·베이크 툴이지만, 모듈화/리팩토링이 안
 된 레거시다. 전역 함수 + maya.cmds 윈도우에 위젯명·콜백을 문자열로 박아 넘기는 옛 방식이라 유지보수가 어렵다.

 이 코드를 JUN_All/tools/A00120_FKIK/ 로 옮겨, 저장소 현행 컨벤션(A00090 / A00110)에 맞춰 UI(app/ui)와 로직(app/core) 분리 +
 PySide(Qt) UI + 셸프 드래그&드롭 설치 구조로 재작성한다.

 사용자 확정 방향

 1. 적극 개선 — 1:1 유지가 아니라 분석 중 발견한 버그를 수정한다(아래 "버그 수정" 절).
 2. 미연결 기능까지 복원 — 원본에 정의됐으나 UI에 안 붙은 기능(토큰 검색, 타입별 Add, parentConstraint 기반 Bake)도 UI에 노출.
 3. 신규 프레임워크 스타일로 현대화 — JUN_/BF_/str_/int_ 접두사 제거, snake_case + 명확한 이름(A00090 ConnectionManager
 스타일). UI 문자는 영어(메모리 규칙 ui-text-english-only).

 모듈별 설계

 app/config/naming.py — ★ 리그 네이밍 데이터 집약 (가장 중요)

 레거시에 흩어진 하드코딩 토큰/이름 리스트를 문자열 그대로 한 곳에 모은다(오타 한 글자가 동작을 깸 → 검증 용이).
 - 상수: BLEND_GRP_TOKEN = "BlendFKIKPos_grp", NUM_POS = 24, POS_SHAPES = ["follicle","locator"], CTL_SHAPE = ["nurbsCurve"]
 - 정렬용 토큰 리스트: TGT_POS_TOKENS(ik+fk), CTL_TOKENS(ik+fk) — JUN_cmd_FKIKTool_setup_btn의 리스트들
 - 매칭 쌍 테이블: FK_PAIRS(tgt↔flw), IK_PAIRS(tgt↔flw) — JUN_matcher_FKIK 의 fk/ik List
 - 부위별 그룹: LIMB_GROUPS = {"arm_l": (tgt[...], flw[...]), "arm_r":..., "leg_l":..., "leg_r":...} — set_match_objs

 app/core/selection_utils.py (← BF_*, JUN_get_*)

 순수/maya.cmds 헬퍼, Qt·위젯 의존 제거:
 - remove_duplicates(items) — list(dict.fromkeys(items)) (순서 유지)
 - hierarchy_without_shapes(obj), make_hierarchy_list(objs, reverse=False, dedupe=True)
 - filter_by_shape_types(objs, shape_types), filter_by_tokens(objs, tokens)
 - order_by_tokens(objs, tokens) — 버그 수정: is "" → 빈 문자열은 if not token: continue
 - move_up(items, indices), move_down(items, indices) — (new_items, new_indices) 반환

 app/core/fkik_setup.py (← JUN_cmd_FKIKTool_setup_btn)

 UI 호출/위젯 갱신을 제거하고 순수 입출력으로:
 - resolve(selection) -> {"ok":bool, "targets":[...], "followers":[...], "message":str}
   - 계층 수집 → BlendFKIKPos_grp 탐색 → pos(follicle/locator) 수집·NUM_POS 검증 → ctl(nurbsCurve)·토큰 필터·NUM_POS 검증 →
 order_by_tokens로 정렬해 반환
   - 실패 사유는 영어 message로(예: "Failed: BlendFKIKPos_grp not found", "Failed: position count mismatch (n/24)")

 app/core/fkik_matcher.py (← match/bake 일체)

 - match_transform(tgt, flw, rot_order=True, rot_axis=True, translate=True, rotate=True) — 단일 쌍 xform 복사(원본
 JUN_MATCH_twoObjects 로직: 임시로 tgt의 rotateOrder 적용 후 복원)
 - class FKIKMatcher:
   - build_pairs(all_targets, all_followers, arm_l, arm_r, leg_l, leg_r, ik_to_fk) -> (tgt_list, flw_list)
       - LIMB_GROUPS로 활성 부위 쌍 모으고, 실제 리스트와 교집합·order_by_tokens, 방향(IK/FK)별 토큰 순 정렬. (레거시의 변수명
 뒤바뀜은 데이터 흐름 유지하되 all_targets/all_followers로 정정)
   - match(tgt_list, flw_list) — 쌍별 match_transform, undo chunk 로 감쌈
   - bake(tgt_list, flw_list, start, end) — 프레임 루프 매칭+setKeyframe. 버그 수정: range(start, end+1)(마지막 프레임 포함),
 undo chunk
   - bake_constraint(tgt_list, flw_list) — 복원: parentConstraint+bakeSimulation(쌍 개수만큼, range(0,8) 하드코딩 → len()
 기반으로 일반화)

 app/core/search_utils.py (← JUN_cmd_Search_By_Token, 복원)

 - search_by_token(items, token, invert=False) -> [matched] — 순수. UI가 선택/하이라이트 처리.

     모듈별 설계

     app/config/naming.py — ★ 리그 네이밍 데이터 집약 (가장 중요)

     레거시에 흩어진 하드코딩 토큰/이름 리스트를 문자열 그대로 한 곳에 모은다(오타 한 글자가 동작을 깸 → 검증 용이).
     - 상수: BLEND_GRP_TOKEN = "BlendFKIKPos_grp", NUM_POS = 24, POS_SHAPES = ["follicle","locator"], CTL_SHAPE =
     ["nurbsCurve"]
     - 정렬용 토큰 리스트: TGT_POS_TOKENS(ik+fk), CTL_TOKENS(ik+fk) — JUN_cmd_FKIKTool_setup_btn의 리스트들
     - 매칭 쌍 테이블: FK_PAIRS(tgt↔flw), IK_PAIRS(tgt↔flw) — JUN_matcher_FKIK 의 fk/ik List
     - 부위별 그룹: LIMB_GROUPS = {"arm_l": (tgt[...], flw[...]), "arm_r":..., "leg_l":..., "leg_r":...} — set_match_objs

     app/core/selection_utils.py (← BF_*, JUN_get_*)

     순수/maya.cmds 헬퍼, Qt·위젯 의존 제거:
     - remove_duplicates(items) — list(dict.fromkeys(items)) (순서 유지)
     - hierarchy_without_shapes(obj), make_hierarchy_list(objs, reverse=False, dedupe=True)
     - filter_by_shape_types(objs, shape_types), filter_by_tokens(objs, tokens)
     - order_by_tokens(objs, tokens) — 버그 수정: is "" → 빈 문자열은 if not token: continue
     - move_up(items, indices), move_down(items, indices) — (new_items, new_indices) 반환

     app/core/fkik_setup.py (← JUN_cmd_FKIKTool_setup_btn)

     UI 호출/위젯 갱신을 제거하고 순수 입출력으로:
     - resolve(selection) -> {"ok":bool, "targets":[...], "followers":[...], "message":str}
       - 계층 수집 → BlendFKIKPos_grp 탐색 → pos(follicle/locator) 수집·NUM_POS 검증 → ctl(nurbsCurve)·토큰 필터·NUM_POS 검증 →
     order_by_tokens로 정렬해 반환
       - 실패 사유는 영어 message로(예: "Failed: BlendFKIKPos_grp not found", "Failed: position count mismatch (n/24)")

     app/core/fkik_matcher.py (← match/bake 일체)

     - match_transform(tgt, flw, rot_order=True, rot_axis=True, translate=True, rotate=True) — 단일 쌍 xform 복사(원본
     JUN_MATCH_twoObjects 로직: 임시로 tgt의 rotateOrder 적용 후 복원)
     - class FKIKMatcher:
       - build_pairs(all_targets, all_followers, arm_l, arm_r, leg_l, leg_r, ik_to_fk) -> (tgt_list, flw_list)
           - LIMB_GROUPS로 활성 부위 쌍 모으고, 실제 리스트와 교집합·order_by_tokens, 방향(IK/FK)별 토큰 순 정렬. (레거시의
     변수명 뒤바뀜은 데이터 흐름 유지하되 all_targets/all_followers로 정정)
       - match(tgt_list, flw_list) — 쌍별 match_transform, undo chunk 로 감쌈
       - bake(tgt_list, flw_list, start, end) — 프레임 루프 매칭+setKeyframe. 버그 수정: range(start, end+1)(마지막 프레임
     포함), undo chunk
       - bake_constraint(tgt_list, flw_list) — 복원: parentConstraint+bakeSimulation(쌍 개수만큼, range(0,8) 하드코딩 → len()
     기반으로 일반화)

     app/core/search_utils.py (← JUN_cmd_Search_By_Token, 복원)

     - search_by_token(items, token, invert=False) -> [matched] — 순수. UI가 선택/하이라이트 처리.

     버그 수정 / 개선 목록 (적극 개선)

     1. order_by_tokens 의 is "" 정체성 비교 → 빈 문자열 falsy 체크(if not token).
     2. bake 프레임 루프 range(start, end) → range(start, end+1) (마지막 프레임 포함).
     3. match/bake/bake_constraint 를 cmds.undoInfo(openChunk/closeChunk) 로 감싸 단일 undo.
     4. 중복 제거를 dict.fromkeys 기반 순서 유지로 통일.
     5. bake_constraint 의 range(0,8) 하드코딩 → 쌍 개수(len) 기반.
     6. match/build_pairs 의 변수명 뒤바뀜(itemsAll_flw/itemsAll_tgt) → 데이터 흐름 유지하되 정확한 이름으로.
     7. 로직에서 UI 직접 갱신(cmds.text/textScrollList) 제거 → core는 값만 반환, UI가 표시.

     ▎ 주의: 리그 네이밍 토큰 문자열은 절대 변경 금지(현장 리그와 1:1). naming.py에 원문 그대로 보존.
