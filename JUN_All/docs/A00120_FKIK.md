# A00120_FKIK 사용법

## 1. 개요

캐릭터 리그의 **FK/IK 매칭 & 베이크**를 돕는 Maya 내 **PySide(Qt) 툴**이다. 선택한 리그
컨트롤에서 팔·다리(좌/우) 타깃/추종 컨트롤을 자동으로 모아, FK↔IK 포즈를 **스냅 매칭**하거나
선택한 **프레임 구간을 베이크**한다.

레거시 maya.cmds/MEL 툴(`PY_JUN_makeUI_FKIKTool`)을 현행 (B) Qt 아키텍처로 리팩토링한 버전이다.

- **로직/UI 분리**: 매칭·베이크 로직은 `app/core`(씬 접근만 `maya.cmds`), 화면은 `app/ui` 가 담당.
  core 는 위젯/컨트롤 이름을 모르고 `list[str]` 만 주고받는다.
- **UI**: PySide(`Framework.qt.qt` 가 PySide6→PySide2 폴백), Maya 메인 윈도우 자식으로 띄운다.
  Targets/Followers 리스트는 재사용 위젯 `Framework.qt.JUN_mod_tsl_qt`(A00140 와 동일).
- **Maya 2023 호환** (Python 3.9 / PySide2). UI 문자열/로그는 모두 영어.

## 3. 설치 / 실행

- **설치**: `A00120_FKIK/__dragDrop_A00120.py` 를 Maya 뷰포트로 **드래그&드롭**하면 현재 셸프에
  버튼이 설치된다.
- **실행**: 셸프 버튼, 또는 스크립트 에디터에서 `tools.A00120_FKIK.run(True)`(`True` 면 reload).

## 4. UI 사용 흐름

1. **Setup** — 리그 컨트롤을 선택하고 **Select Objects** → `FKIKSetup.resolve` 가 선택에서
   Targets/Followers 를 자동 해석해 두 리스트를 채운다. 상태 라벨(녹색=성공)로 결과 표시.
2. **Limb** — Arm Left/Right · Leg Left/Right 체크로 매칭·베이크 대상 사지를 한정한다(기본 전체 On).
3. **Targets / Followers** — 재사용 tsl 위젯(Select/Add/Del/Up/Down/Sort, 씬 선택 연동).
4. **Search** — token 입력 + (Invert) 로 두 리스트에서 일치 항목을 선택(씬에서도 select).
5. **Match / Bake**
   - **Match IK / Match FK** : 현재 프레임에서 한 번 포즈 스냅(`build_pairs` → `match`).
   - **Bake IK / Bake FK** : Start–End 구간을 임시 `parentConstraint` + `bakeResults` 로 베이크.
   - **Bake (Constraint)** : 양쪽 리스트에서 **선택한 쌍**을 재생 범위 전체에 걸어 `bakeSimulation`.
6. **Log / Force Refresh** — 작업 로그 표시. 뷰포트가 멈춘(커브 편집이 프레임 이동 전까지 안 보임)
   드문 경우 **Force Refresh** 로 막힌 전역 refresh suspend 를 풀고 한 번 다시 그린다.

---

## 5. 베이크 동작 상세

`FKIKMatcher.bake(tgt, flw, start, end)` 와 `bake_constraint(tgt, flw)` 는 각 쌍에 임시
`parentConstraint` 를 걸어 follower 가 target 의 월드 포즈를 따라가게 한 뒤 한 번에 샘플링하고,
베이크 후 컨스트레인트를 삭제한다. 대량 키 작업은 `suspend_refresh()` 로 감싸 프레임마다
리드로우하지 않는다.

### 구간 밖 키 보존 (v01.05~)

구간(`[start, end]`)만 베이크해도 그 **바깥의 기존 키가 모두 사라지던 버그**가 있었다.
원인은 임시 `parentConstraint` 를 거는 순간 Maya 가 `pairBlend` 를 끼워 기존 `animCurve` 를
플러그에서 분리하기 때문이다 — 이러면 `bakeResults` 의 `preserveOutsideKeys=True` 가 바깥 키를
"보지 못해" 보존할 수 없다.

해결: 컨스트레인트를 걸기 **전에** `[start, end]` 밖의 키를 값/탄젠트(타입, fixed 면 각도/가중치까지)
단위로 **스냅샷**(`_snapshot_outside_keys`)하고, 베이크·컨스트레인트 정리 **후에 다시 찍어 복원**
(`_restore_outside_keys`)한다. 베이크 구간만 새 키로 대체되고 나머지 애니메이션은 그대로 유지된다.
`bake_constraint()` 도 재생 범위 밖 키에 동일한 가드를 적용한다.

> 예: 0~1000 프레임에 키가 있고 500~600 만 Bake → 0~500, 600~1000 의 원본 키가 그대로 남는다.
