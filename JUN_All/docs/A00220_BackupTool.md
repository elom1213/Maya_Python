# A00220_BackupTool — 사용 안내

컴퓨터가 사고로 종료되는 상황에 대비해, **지정한 파일들을 일정 주기마다 자동으로 복사·백업**하는
파이프라인 보조 툴. 작업 파일을 옆에 계속 복제해 두어 **갑작스러운 종료/크래시 시 직전 상태를
복구**할 수 있게 한다.

> 이 툴은 Maya 안에서 도는 툴이 아니라 `A00210_FileManager` 처럼 **Windows 에서 독립 실행되는
> PySide6 앱** 이다. Maya 설치/실행 없이 동작한다.

---

## 1. 핵심 개념

| 개념 | 설명 |
|------|------|
| **Target Files** | 백업할 파일 목록. 여러 개를 한 번에 등록할 수 있다. |
| **backup 폴더** | 백업본이 쌓이는 곳. **각 원본 파일이 있는 폴더 아래**에 만들어진다(폴더가 이미 있으면 그대로 사용). 폴더 이름은 UI 에서 지정(기본 `backup`). |
| **Suffix** | 백업 파일 이름 뒤에 붙는 표식(기본 `BU`). `scene.mb` → `scene_BU.mb`. |
| **Save Mode** | `Overwrite`(기본) 또는 `Version Up`. |
| **Interval** | 저장 주기(분 + 초). |
| **Auto Backup** | 켜면 **파일을 저장한 뒤 `Save Delay` 초 후에** 그 파일만 백업한다. 끄면 매 `Interval` 마다 전 파일을 백업. |
| **Save Delay** | Auto Backup 전용. 저장 감지 후 실제 백업까지의 **지연(초, 기본 10)**. 크래시로 손상된 파일이 정상 백업본을 덮어쓰는 사고를 막는 settle 시간(아래 5-1 참고). |

백업 파일은 확장자 **앞**에 접미사를 붙인다: `scene.mb` → `scene_BU.mb`(덮어쓰기) /
`scene_BU_01.mb`, `scene_BU_02.mb` …(버전업).

설정(파일 목록·옵션·주기)은 git 으로 공유하지 않고 **로컬 prefs** 에 저장된다:
`%USERPROFILE%/.jun_backuptool/prefs.json` (다음 실행 시 자동 복원).

## 2. 실행

- **개발 실행**: `python JUN_All/tools/A00220_BackupTool/launch.py`
- **exe 빌드**: 툴 폴더의 `build_exe.bat`(PyInstaller, `launch.spec`) → `dist/A00220_BackupTool.exe`
- 필요 패키지: `PySide6`, `pyinstaller`. 백업 로직은 **표준 라이브러리**(shutil/os/re/json)만 쓴다.

## 4. 저장 모드

| 모드 | 동작 | 결과 파일 |
|------|------|-----------|
| **Overwrite** (기본) | 매 주기마다 같은 이름으로 **덮어쓴다**. | `scene_BU.mb` 하나만 계속 갱신 |
| **Version Up** | 매 주기마다 번호를 올려 새 파일을 만든다. **최근 N개만 유지**(N 초과 시 가장 오래된 버전 자동 삭제 = 롤오버). `Max Versions` 로 N 지정. | `scene_BU_01.mb`, `scene_BU_02.mb` … |

## 5. 상태 표시 — Chrome Dino (v01.04, 회전 v01.06, 저장 톡 점프 v01.09)

`Control` 의 상태는 **글자 대신 Chrome-Dino(T-Rex) 픽셀 애니메이션**으로 보인다
(이미지 에셋 없이 `app/ui/dino_widget.py` 가 QPainter 로 그린다 — 테마 무관, exe 번들 불필요).

| 공룡 동작 | 의미 |
|-----------|------|
| **가만히 서 있음** | 정지 상태(기본, Deactive). |
| **제자리 달리기 + 주기적 점프** | 백업 동작 중(Active). 다리가 교차하고 바닥 점선이 흐르며 약 2.6초마다 (회색·높은) 점프. |
| **강조색 톡 점프(notify_save)** | **사용자가 대상 파일을 저장한 순간**(디스크 변경 감지) 짧고 낮게 한 번 톡 뛰며 **빨간색**으로 반짝인다(v01.09; 색을 고정 빨강 `#E53935` 으로 통일 v01.11). 회색 자동 점프보다 **짧고 낮고 색이 달라** 구분된다. **Auto Backup 모드에서만** 나타난다(주기 모드는 저장 순간을 알 수 없음). |
| **공중에서 360° 회전(spin)** | **파일이 실제로 백업된 순간** 한 바퀴 돌아 백업 성공을 또렷이 알린다(v01.06). 대상 파일 중 하나라도 백업되면 트리거. |
| `Next save in  MM:SS` | 다음 자동 저장까지 남은 시간(1초마다 갱신). 정지 시 `--:--`. |

> **저장 순간 vs 백업 순간**: Save Delay(v01.08) 로 두 순간이 수 초 벌어지므로, **저장 감지 = 강조색 톡 점프**,
> **실제 백업 = 360° 스핀**으로 나눠 눈으로 구분한다. 우선순위는 스핀 > 저장 톡 점프 > 점프 > 자동 점프라,
> 저장 톡 점프 도중 백업 스핀이 시작되면 스핀이 우선한다.
>
> v01.06 이전에는 저장 순간 작은 점프(hop) 한 번이라 놓치기 쉽고, 실제로 복사되지 않아도(변경 없음·스킵)
> 뛰던 문제가 있어 **실제 백업 성공(`ok > 0`) 시 360° 회전**으로 바꿨다(그 hop 은 v01.09 에서 강조색 저장
> 톡 점프로 부활). 회전하는 동안 스프라이트 모서리가 잘리지 않도록 위젯 세로 여유를 대각선만큼 확보하고,
> 회전 중에는 주기 점프를 멈춘다.

## 5-1. Auto Backup — 저장 후 지연 백업 (v01.05, 지연 v01.08, 감지 폴링 v01.10)

`Control` 의 **Auto Backup** 체크박스(기본 켜짐)는 백업을 *주기*가 아니라 *저장 시점*에 맞춘다.

| Auto Backup | 동작 |
|-------------|------|
| **켜짐**(기본) | 대상 파일을 실시간 감시(`QFileSystemWatcher` + mtime 폴링)하다가 **저장이 감지되면 `Save Delay` 초 뒤에** 그 파일만 백업한다. 바뀌지 않은 파일은 건드리지 않는다. 카운트다운 대신 `Auto save` 표시. |
| **꺼짐** | 매 `Interval` 마다 **전 파일**을 백업(고전적 주기 백업). |

- `Interval` 타이머는 Auto Backup 이 켜져 있어도 **감시가 놓친 변경을 잡는 fallback** 으로 계속 돈다.
- 실행 중에 Auto Backup 을 켜고/끄면 감시도 즉시 시작/정지된다.
- **저장 감지 방식(v01.10)**: `QFileSystemWatcher` 는 **Windows 10** 이나 임시파일 교체식(atomic
  replace) 저장에서 안정적으로 발화하지 않아, 저장 순간(공룡 톡 점프·저장 감지 백업)을 놓칠 수 있었다.
  이를 보완하려고 감시 중에는 **0.5초 mtime 폴링**(`_save_poll_timer`)을 함께 돌려 OS/저장 방식과
  무관하게 저장을 잡는다. 감시자와 폴링은 공통 처리(`_on_save_detected`)로 모이며 **mtime 으로 중복
  발화를 제거**하므로, 감시자가 정상 발화하는 Windows 11 에서도 톡 점프는 정확히 한 번만 재생된다.
- **저장 톡 점프 색(v01.11)**: 저장 순간 강조색을 예전엔 OS 팔레트 하이라이트(`palette().highlight()`)에서
  가져왔는데, 이 색이 **Windows 버전마다 달라**(Win11=빨강 계열, Win10=시스템 파랑 등) Win10 에서는
  톡 점프가 빨갛게 안 나왔다. 이제 OS/테마 팔레트에 기대지 않고 **고정 빨강 `#E53935`** 으로 직접 그려
  모든 OS 에서 동일하게 빨간 톡 점프가 보인다(점프 타이밍·높이는 그대로).

### 왜 '지연'인가 — 크래시로 손상된 파일 백업 방지 (v01.08)

이 PC 는 사고로 자주 종료되는데, **종료 시점에 저장 중이던 Maya 파일이 손상된 채 디스크에 남는다.**
예전(v01.05~01.07)처럼 저장을 감지하자마자(~300ms) 백업하면, 그 손상된 파일이 **정상 백업본을 덮어써**
백업까지 못 쓰게 되는 사고가 났다.

v01.08 은 저장을 감지해도 곧바로 복사하지 않고, **파일별로 `(지금 + Save Delay)` 시점에 백업을 예약**한다.

- 정상 저장(Ctrl+S) → `Save Delay` 초 뒤 정상 백업(조금 늦을 뿐 무해).
- **저장 직후 PC 가 크래시로 종료** → 예약 백업은 이 툴의 in-process 타이머라 **프로세스와 함께 사라진다**
  → 손상된 파일이 백업되지 않고 **직전의 정상 백업본이 그대로 보존**된다.
- 지연 중 같은 파일이 다시 저장되면 그 파일만 `마지막 저장 + Delay` 로 재예약(settle)한다.
- 예약은 1초마다 폴링되며(예약이 있을 때만 동작), 여러 파일은 각자 독립적으로 지연·백업된다.

> **핵심 원리**: 백업 툴이 크래시하는 PC 와 **운명을 함께한다**. "저장 후 N초간 시스템이 살아있었다
> = 정상 저장이었을 가능성이 높다" 는 settle 휴리스틱이다. 손상 여부를 검사하는 것이 아니라, 위험한
> 구간(저장 직후)을 통과할 때까지 백업을 미뤄 크래시가 백업을 무산시키게 한다.
>
> **보완 팁**: `Version Up` 모드를 함께 쓰면, 만에 하나 손상본이 백업돼도 이전 정상 버전들이 남아
> 이중 안전망이 된다.

## 6. 사용 순서

1. `Add Files...` 로 백업할 파일을 등록한다(여러 개 가능).
2. `Backup Folder Name` / `Suffix` 를 원하면 바꾼다(기본 `backup` / `BU`).
3. `Save Mode` 선택 — 보통 `Overwrite`. 이력을 남기려면 `Version Up` + `Max Versions`.
4. `Auto Backup`(기본 켜짐) — 저장 후 지연 백업할지(켜짐), 주기마다 전 파일을 백업할지(꺼짐) 고른다.
   켜면 `Save Delay`(기본 10초) 로 저장 후 백업까지의 지연을 정한다(크래시 손상 방지, 5-1 참고).
5. `Interval` 에 주기(분·초)를 넣는다. Auto Backup 이 켜져 있으면 fallback 주기로 쓰인다.
6. `Start` — 즉시 1회 백업 후, 저장할 때마다(또는 주기마다) 반복.
7. `Stop` — 백업 중지(`Deactive`). 창을 닫으면 타이머·감시가 멈추고 설정이 저장된다.

> 백업 도중 원본 파일이 사라지면 그 파일만 건너뛰고(로그에 표기) 나머지는 계속 백업한다.

## 7. 구조 (개발 참고)

```
A00220_BackupTool/
├── launch.py                 # standalone 진입점 (green_dark 테마)
└── app/
    ├── config/version.py     # VERSION / LAST_UPDATE
    ├── core/
    │   ├── backup_manager.py # 백업 핵심 로직 (Qt/DCC 비의존)
    │   └── prefs.py          # 로컬 설정 영속화
    └── ui/main_window.py     # Qt UI + 타이머(주기 백업·저장후 지연 백업 예약·카운트다운)
```

- `app/core`(로직)와 `app/ui`(화면)를 분리한다(`backup_manager` 는 표준 라이브러리만 사용).
- import 는 **툴 고유 경로**(`tools.A00220_BackupTool.app...`)와 내부 **상대 import** 로 한다 —
  여러 standalone 툴을 한 인터프리터에서 동시에 띄울 때 최상위 `app` 패키지가 충돌하지 않도록.
