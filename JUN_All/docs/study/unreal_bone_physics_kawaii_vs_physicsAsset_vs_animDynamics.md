# 언리얼 본 피직스 3종 동작 원리 — Kawaii Physics vs Physics Asset vs Anim Dynamics

> **주제**: 언리얼에서 아바타의 본(머리카락·치마·꼬리·가슴 등)에 세컨더리 모션(흔들림)을 주는
> 세 가지 방식의 **내부 동작 원리**와, 왜 **Physics Asset** 만 유독 "리얼"하게 느껴지는지
> (중력·컬라이더 반응이 왜 더 엄밀한지)를 정리한다.
>
> 도메인: 리깅 / 시뮬레이션 / 언리얼 애니메이션.
> 관련 툴: [A00080_KWI_creator](../A00080_KWI_creator_V03.md)(KawaiiPhysics 노드 텍스트 생성기).

---

## 0. 세 줄 요약 (먼저)

- **Kawaii Physics** = AnimGraph 안에서 도는 **절차적(가짜) 본 스프링**. 본 *끝점 위치*만 Verlet 로 풀고
  회전은 look-at 으로 유도. 질량·관성·진짜 충돌이 없다 → 싸고 안정적이지만 "부드럽게 늘어지는" 느낌.
- **Anim Dynamics** = AnimGraph 안에서 도는 **경량 강체 시뮬**. 본마다 박스 강체 + 관절 구속. 관성텐서·각운동량이
  있어 Kawaii 보다 진자 운동이 리얼하지만 충돌은 여전히 단순(평면/구).
- **Physics Asset** = **진짜 물리 엔진(Chaos/PhysX)** 이 6-DOF 강체를 고정 서브스텝으로 적분.
  질량·관성텐서·양방향 접촉·마찰·월드 충돌까지 엄밀 → **가장 리얼**(하지만 가장 비싸고 튈 수 있음).

## 1. 공통 배경 — "본에 흔들림을 준다"는 같은 문제, 두 갈래의 해법

세 방식 모두 목표는 같다: **애니메이션이 이미 만든 포즈(pose) 위에, 관성으로 인한 늦게 따라오는 흔들림을
덧씌운다.** 차이는 *그 흔들림을 어떻게 계산하느냐* 이고, 크게 두 계열로 갈린다.

| 계열 | 어디서 도나 | 뭘 푸나 | 대표 |
|------|-------------|---------|------|
| **AnimGraph 절차적/경량 솔버** | 애니메이션 평가(anim thread), 포즈 계산 중 | 본을 직접 근사 계산 | Kawaii Physics, Anim Dynamics |
| **진짜 물리 엔진** | 물리 씬(physics scene), 고정 서브스텝 | 6-DOF 강체 + 구속 정식 해석 | Physics Asset (Ragdoll / RigidBody 노드) |

> **핵심 통찰**: "리얼함"의 차이는 대부분 **어느 계열에서 도느냐**에서 나온다. 절차적 솔버는
> *안정성·성능·아트 컨트롤* 을 위해 물리 법칙을 의도적으로 근사/생략한다. 물리 엔진은 반대로
> *물리적 정확성* 을 위해 무겁고 예측 불가능함을 감수한다.

UE 버전 메모: 물리 엔진은 UE4 = **PhysX**, UE5 = **Chaos**. Anim Dynamics·RigidBody 노드는 **엔진 내장**,
Kawaii Physics 는 **서드파티 플러그인**(pafuhana1213 / Yuya Ura, 오픈소스).

## 2. Kawaii Physics — 절차적 본 스프링 (Verlet + 거리 구속)

**한 문장**: 본의 *끝점(tip) 위치* 하나만 Verlet 적분으로 흔들고, 본 길이는 딱딱한 거리 구속으로 유지,
회전은 흔들린 끝점을 바라보게(look-at) 되짚어 만든다.

### 2.1 매 업데이트 절차 (개념)
각 시뮬 본은 `Location`, `PrevLocation`, `PoseLocation`(애니메이션이 준 원래 위치)을 들고 있다.

```
1) 애니메이션 포즈에서 각 본의 PoseLocation 을 읽는다.
2) Verlet 적분:  velocity = (Location - PrevLocation) * (1 - WorldDamping)
                 Location += velocity + (Gravity + Wind + ExternalForce) * dt^2
3) Limit 적용:   Spherical / Capsule / Planar 리미트 = 점을 도형 밖으로 밀어냄(기하 투영)
4) Stiffness:    PoseLocation 쪽으로 다시 끌어당김(포즈 복원력)
5) 거리 구속:    부모까지 거리 = 원래 본 길이가 되도록 위치 보정 (딱딱한 막대)
                 + LimitAngle: 포즈 방향에서 너무 벗어나지 않게 원뿔 안으로 클램프
6) 회전 유도:    부모→새 Location 방향으로 본의 forward 축을 정렬(look-at)해 회전 계산
```

### 2.2 성질 (왜 이렇게 느껴지나)
- **시뮬 대상이 "점"이다.** 본 끝 위치만 푼다. 회전은 결과물이지 상태가 아니다 → **트위스트(비틀림) 불가**,
  사실상 자유도 1 수준.
- **질량·관성 개념이 없다.** `Stiffness`, `Damping`, `Gravity` 는 물리량이 아니라 *스프링 계수* 다.
  무거운 것이 천천히, 가벼운 것이 팔랑이는 차등이 자동으로 안 나온다(파라미터로 흉내낼 뿐).
- **강하게 감쇠(damping)** 되고 거리·각도 구속에 즉시 눌려서, 흔들림이 평형점을 크게 지나쳤다 되돌아오는
  *오버슛/진동* 이 약하다 → "탄력 있는 흔들림"보다 "부드럽게 늘어졌다 멈추는" 느낌.
- **매우 싸고 완전히 안정적이며 결정론적.** 폭발(explode)하지 않는다. LOD·양산 캐릭터·스타일라이즈드에 이상적.

### 2.3 컬라이더(Limit)의 정체
Spherical / Capsule / Planar **리미트**는 물리 씬의 콜라이더가 아니라 노드에 정의된 **해석적 도형**이다.
동작은 "이 점이 도형 안에 들어왔으면 표면 밖으로 밀어낸다"는 **한 방향 위치 투영**뿐. → 마찰 없음,
접촉력(반작용) 없음, 임의의 월드 지오메트리와는 충돌 못 함.

## 3. Anim Dynamics — AnimGraph 안의 경량 강체 시뮬

**한 문장**: 본(또는 체인 링크)마다 **박스 강체 하나 + 부모와의 관절 구속**을 만들어, 실제로 선속도·각속도를
적분하는 *작은 물리 시뮬* 을 AnimGraph 안에서 돌린다.

### 3.1 원리
- 각 본을 **박스 강체**로 본다. 박스의 크기(Box Extents)가 **관성텐서(inertia tensor)** 를 결정한다.
- 중력이 강체의 무게중심(관절축에서 오프셋된 위치)에 작용 → **토크(torque)** 발생 → 진짜 진자처럼 흔들린다.
- **선속도 + 각속도를 적분**하므로 각운동량(angular momentum)·오버슛·잔진동이 물리적으로 나온다.
- 구속: Linear Limit(선형 한계, 스프링/하드), Angular Limit(원뿔·트위스트 한계), 선택적 타겟 스프링.
- 충돌: Planar / Spherical 리미트(제한적). 여기까지는 Kawaii 와 비슷한 단순 도형.
- 서브스텝(MaxSubstep) 지원 → dt 를 잘게 쪼개 안정화.

### 3.2 Kawaii 와의 결정적 차이
Kawaii 가 "점"을 푸는 반면 Anim Dynamics 는 **관성텐서를 가진 강체**를 푼다. 그래서
**중력에 의한 토크·각운동량이 실제로 존재** → 진자 운동, 관성으로 인한 지연·오버슛이 훨씬 리얼하다.
다만 충돌은 여전히 단순 도형이고, 전용 경량 솔버라 Physics Asset 만큼 엄밀하진 않다.

## 4. Physics Asset — 진짜 물리 엔진의 6-DOF 강체 시뮬

**한 문장**: 본마다 **콜리전 지오메트리(질량 포함) 강체(Body)** 를, 본 사이마다 **6-DOF 관절(Constraint)** 을
정의한 데이터 자산을, **물리 엔진(Chaos/PhysX)** 이 고정 타임스텝으로 엄밀하게 시뮬레이션한다.

### 4.1 Physics Asset 을 이루는 것
- **Body (BodySetup)**: 본에 붙은 콜리전 프리미티브(Sphere/Capsule/Box/Convex) + **질량·밀도** + 피지컬 머티리얼.
  부피와 밀도로부터 **질량과 전체 관성텐서**가 계산된다.
- **Constraint (ConstraintInstance)**: 두 Body 를 잇는 **6자유도 관절**. Linear/Angular 한계, **드라이브(위치·속도
  스프링 모터)**, 목표 지향 등.

### 4.2 어떻게 시뮬되나 (두 경로)
1. **Ragdoll (월드 물리 씬)**: `SetSimulatePhysics` 로 전신/부분을 월드 씬에서 완전 시뮬레이션. 캐릭터가 쓰러지는 등.
2. **RigidBody AnimGraph 노드 (immediate mode)**: 같은 Chaos/PhysX 솔버를 **AnimGraph 안에서** 돌려,
   Physics Asset 을 세컨더리 모션(머리카락·꼬리·가슴·천)에 쓰는 현대적 방식. 솔버는 진짜지만 평가는 포즈와 함께.

두 경로 모두 **진짜 물리 엔진**을 쓴다는 게 핵심. 그래서 리얼하다.

### 4.3 물리 엔진이 하는 일 (Kawaii/AnimDynamics 가 생략하는 것)
- **고정 서브스텝 적분(substepping)**: 프레임레이트와 분리된 고정 dt 로 수치 적분 → 중력을 *진짜 가속도* 로
  안정·정확하게 적분.
- **질량 + 전체 관성텐서**: 무거운 건 천천히, 형상에 따라 회전 저항이 다르게, 토크가 정확히.
- **반복 구속 솔버(iterative solver)** + **접촉 처리**: 브로드페이즈→내로우페이즈 충돌 검출, 접촉 매니폴드,
  침투 복원, **마찰·반발(restitution)**, **양방향** 반작용.
- **체인 전체를 커플링해서 한 번에 해석** → 에너지가 관절을 타고 물리적으로 전파.

## 5. ★핵심★ 왜 Physics Asset 이 더 "리얼"한가 — 5개 축

사용자가 느낀 "중력 표현이 리얼하고, 컬라이더 반응이 더 엄밀하다"는 것은 아래 5가지가 동시에 작용한 결과다.

### 5.1 중력 — "위치 넛지" vs "진짜 가속도 적분"
- **Kawaii**: 중력은 그저 방향 벡터 파라미터. 매 프레임 `Gravity * dt²` 만큼 끝점을 살짝 밀지만,
  즉시 **강한 거리·각도 구속과 감쇠에 눌린다.** 결과적으로 **동적인 "낙하"가 아니라 정적인 "처짐(droop)"** 에
  가깝다. 관성으로 평형점을 지나쳤다 되돌아오는 맛이 약하다.
- **Anim Dynamics**: 중력이 무게중심에 토크를 만들어 **진짜 진자처럼** 흔들린다(중간 단계).
- **Physics Asset**: 중력을 `g ≈ 980 cm/s²` 의 **실제 가속도**로 고정 서브스텝에서 적분. 물체가
  **가속→운동량 축적→평형점 통과→감쇠 진동→정착**하는 실제 낙하/스윙 곡선을 그린다.
  → 이게 "중력이 리얼하다"의 정체.

### 5.2 질량·관성 — 없음 vs 근사 vs 정식
- **Kawaii**: 질량·관성텐서 **없음**. Stiffness/Damping 은 스프링 계수일 뿐. 무게 차등이 물리적으로 안 나옴.
- **Anim Dynamics**: 박스 크기로 **관성텐서 근사** → 각운동량 있음(리얼함↑).
- **Physics Asset**: 콜리전 형상+밀도로 **질량·전체 관성텐서 정식 계산** → 토크·회전 저항·무게감이 정확.

### 5.3 컬라이더 반응 — "한 방향 밀어내기" vs "진짜 접촉 해석"
사용자가 느낀 두 번째 포인트. 이게 가장 크게 갈린다.

| | Kawaii / Anim Dynamics 의 Limit | Physics Asset 의 Collision |
|---|---|---|
| 콜라이더 정체 | 노드에 적은 **해석적 도형**(구/캡슐/평면) | **물리 씬의 실제 콜리전 프리미티브** |
| 처리 방식 | 점을 표면 **밖으로 위치 투영**(한 방향) | 접촉 매니폴드 기반 **양방향 반작용** |
| 마찰 / 반발 | 없음 | **있음**(피지컬 머티리얼) |
| 침투 복원 | 단순 밀어내기 | 반복 솔버로 정밀 복원(관통 방지) |
| 월드/타 강체와 충돌 | **불가**(정의된 도형에만) | **가능**(월드 지오·다른 Body·셀프 콜리전) |

→ Kawaii 는 "이 구 안에 들어오면 밖으로 밀어내"가 전부라 **표면을 타고 미끄러지거나(마찰), 부딪혀 튕기거나,
여러 콜라이더 사이에 끼는** 엄밀한 반응이 안 나온다. Physics Asset 은 접촉을 **힘/충격량**으로 풀기 때문에
자연스럽게 나온다. → 이게 "컬라이더 반응이 엄밀하다"의 정체.

### 5.4 자유도(DOF)·구속 — 1-DOF look-at vs 6-DOF 관절
- **Kawaii**: 끝점 위치만 풀고 회전은 look-at 으로 유도 → **트위스트 불가**, 사실상 1자유도.
- **Physics Asset**: 각 관절이 **6-DOF**(3 병진 + 3 회전) 한계·드라이브를 가짐 → 비틀림, 정확한 관절 한계,
  스프링 모터까지. 복잡한 이차 운동을 표현.

### 5.5 커플링 솔버 — 본별 완화 vs 체인 동시 해석
- **Kawaii**: 본을 위→아래로 훑으며 거리 구속을 몇 번 완화(relaxation)하고, 안정성 위해 강하게 감쇠 →
  에너지가 손실되어 덜 생동감.
- **Physics Asset**: 체인 전체를 하나의 연립계로 반복 해석 → 에너지가 관절을 타고 물리적으로 전파,
  살아있는 흔들림.

> **결론**: Kawaii Physics 가 덜 리얼한 건 "성능이 낮아서"가 아니라, **안정성·성능·아트 컨트롤을 위해
> 위 5가지(진짜 적분·질량/관성·접촉 해석·6-DOF·커플링)를 일부러 근사하거나 생략**했기 때문이다.
> Physics Asset 은 그 5가지를 전부 물리 엔진에 맡겨 "리얼함"을 얻고, 대신 비용·불안정성·낮은 아트 컨트롤을
> 치른다.

## 6. 한눈 비교표

| 축 | Kawaii Physics | Anim Dynamics | Physics Asset |
|---|---|---|---|
| 계열 | 절차적 본 스프링 | 경량 강체 시뮬 | 진짜 물리 엔진(Chaos/PhysX) |
| 시뮬 대상 | 본 끝점(위치) | 박스 강체 | 6-DOF 콜리전 강체 |
| 적분 | Verlet(위치 넛지) | 선/각속도 적분 | 고정 서브스텝 정식 적분 |
| 질량/관성 | 없음 | 박스 근사 | 질량+전체 관성텐서 |
| 회전 | look-at 유도(트위스트✕) | 강체 각속도 | 6-DOF 관절 |
| 중력 | 정적 처짐 | 진자(토크) | 실제 가속도 낙하/스윙 |
| 충돌 | 도형 한 방향 밀어내기 | 단순 도형 | 접촉·마찰·반발·월드/셀프 |
| 실행 위치 | AnimGraph(anim thread) | AnimGraph | 물리 씬 / RigidBody 노드 |
| 비용 | 매우 낮음 | 낮음~중간 | 높음 |
| 안정성 | 폭발 없음, 결정론적 | 대체로 안정 | 튈 수 있음 |
| 아트 컨트롤 | 매우 좋음 | 좋음 | 낮음("물리가 주는 대로") |
| 리얼함 | ★ | ★★ | ★★★ |

---

## 7. 그럼 언제 뭘 쓰나 (트레이드오프)

- **Kawaii Physics** — 머리카락·치마·꼬리·귀·가슴 등 **스타일라이즈드/양산 세컨더리**. 무조건 안정적이고 싸며
  아트 디렉션(클램프·과장)이 쉬워야 할 때. 카툰 룩과 궁합 최고.
- **Anim Dynamics** — Kawaii 보다 **관성감 있는 진자 운동**이 필요하지만 풀 물리는 과할 때의 절충. 내장 노드.
- **Physics Asset (RigidBody 노드)** — **리얼리스틱** 캐릭터에서 중력·충돌이 물리적으로 정확해야 하고 비용을
  감당할 수 있을 때. 래그돌·정밀 천/살결 흔들림.
- 실무에선 섞어 쓴다: 클로즈업/주역은 Physics Asset, 원거리·군중은 LOD 로 Kawaii/AnimDynamics.

> 성능·안정성이 필요하면 절차적 솔버, 물리적 정확성이 필요하면 물리 엔진. **"리얼함 = 물리 엔진에
> 얼마나 맡겼는가"** 로 이해하면 세 방식이 한 스펙트럼 위에 놓인다.

---

## 8. 한 줄 요약

> **Kawaii(점 스프링) → Anim Dynamics(경량 강체) → Physics Asset(풀 물리 엔진)** 순으로
> "물리 엔진에 맡긴 정도"가 커지고, 그만큼 **중력이 진짜 가속도로 적분되고 충돌이 양방향 접촉으로 해석**되어
> 리얼해진다. Physics Asset 의 리얼함은 **질량·관성텐서·고정 서브스텝 적분·6-DOF 구속·접촉 마찰**을 전부
> 갖췄기 때문이고, Kawaii 가 덜한 건 성능·안정성·아트 컨트롤을 위해 그것들을 **의도적으로 근사/생략**했기 때문이다.
