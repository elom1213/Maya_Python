# 버텍스 노말 색(노랑/초록)의 의미와 FBX 추출 시 커스텀 노말 보존 — 학습 노트

> **주제**: Maya `Display → Polygons → Vertex Normals` 로 노말을 볼 때 나타나는 **노랑/초록** 두 색의
> 차이(= locked vs unlocked 노말)와, FBX 로 추출할 때 **노랑(커스텀) 노말이 사라지고 초록만 남는 현상**의
> 원인·해결.
>
> 도메인: 모델링 / 셰이딩 / 파이프라인(FBX 익스포트).

---

## 1. 노랑 vs 초록 노말의 차이

Maya 의 Vertex Normals 표시에서 **색은 그 노말이 잠겨(locked) 있는지** 여부를 나타낸다.

| 색 | 의미 | 성격 |
|----|------|------|
| **초록 (green)** | **Unlocked** 노말 | Maya 가 **자동 계산**한다. 면 각도·소프트/하드 엣지·스무딩에 따라 그때그때 평균내는 값으로, 메시를 편집하면 다시 계산된다. |
| **노랑 (yellow)** | **Locked** 노말 | 사용자가 **명시적으로 고정한 값(커스텀 노말, explicit normal)**. Set Normal / Lock Normals / 노말 전사(transfer) 등으로 박아넣은 값이며, Maya 가 임의로 다시 계산하지 않는다. |

- 즉 **노랑 = 의도적으로 만든 커스텀 노말**, **초록 = Maya 가 알아서 계산하는 기본 노말**.
- 보통 셰이딩을 손으로 다듬은 부분(식물 잎, 헤어 카드, 둥글게 보이게 한 하드서피스 등)이 노랑이다.

> 참고: "locked/unlocked"(노말 값 고정 여부)와 "hard/soft edge"(노말을 엣지에서 나눌지 평균낼지)는
> 서로 관련되지만 **다른 개념**이다. 색이 나타내는 건 **lock 상태**다.

---

## 2. 왜 FBX 에서 노랑(커스텀)이 사라지나

핵심: **unlocked 노말은 "확정된 데이터"가 아니라서**, 받는 쪽이 다시 계산하면 커스텀 값과 섞이거나
덮어써진다. 원인은 보통 두 군데다.

1. **Export 쪽**: 노말이 일부만 잠겨 있으면(노랑+초록 혼재) FBX 가 노말을 스무딩 그룹/엣지 기준으로
   **재생성 가능한 상태**로 내보내고, 임포터가 다시 계산하면 커스텀 값이 날아간다.
2. **Import 쪽 (진짜 원인인 경우가 많음)**: 받는 엔진/DCC 의 **노말 임포트 옵션이 "다시 계산
   (Compute / Calculate)"** 으로 되어 있으면, FBX 안에 노말이 들어 있어도 무시하고 스무딩으로 새로
   만들어 → 항상 초록(기본 계산값)만 보인다.

---

## 3. 노랑 노말을 살려서 추출하는 방법

### (A) Export 전 — 노말을 전부 Lock 해서 explicit 데이터로 만들기
1. 메시 선택
2. **Mesh Display → Lock Normals**
   - 현재 보이는 노말(초록 계산값 포함)이 **전부 그 값 그대로 고정(노랑)** 된다 → FBX 에 **모든 노말이
     명시적 데이터로** 기록된다.
   - 의도한 셰이딩이 지금 화면에 맞다면, 이 상태로 잠그면 그 모습이 그대로 박힌다.
3. (필요시) 하드/소프트 엣지를 먼저 정리한 뒤 Lock 한다. **Lock 은 "지금 모습"을 굳히는 것**이라,
   잠그기 전 셰이딩이 맞아야 한다.

### (B) FBX Export 설정
- **Geometry → Smoothing Groups: ON**, **Tangents and Binormals: ON**(노말과 함께 탄젠트도 보존하려면).
- Normals 는 기본적으로 export 되지만, (A) 처럼 잠가두면 **explicit normal** 로 나간다.

### (C) Import 쪽 옵션 — 보통 여기가 결정적
- **Unreal**: FBX Import → Mesh → **Normal Import Method = `Import Normals`**(또는
  `Import Normals and Tangents`). 기본값 `Compute Normals` 면 무조건 재계산되어 커스텀 노말이 사라진다.
- **Unity**: Model Import → **Normals = `Import`**(Calculate 아님).
- 다른 DCC 로 재임포트해 확인할 때도 "노말 다시 계산" 옵션을 끈다.

---

## 4. 요약

- **노랑 = 잠긴 커스텀 노말(의도한 값)**, **초록 = Maya 자동 계산값**.
- 살리는 핵심: **Export 전에 `Lock Normals` 로 전부 명시 데이터화** + **받는 쪽 임포트 옵션을
  `Import Normals`(재계산 끄기)** 로 설정.
- **확인 팁**: Lock 후엔 메시의 모든 노말이 노랑으로 바뀐다. 그 상태로 내보내고 임포트 옵션을
  Import Normals 로 두면 양쪽 셰이딩이 일치한다.
