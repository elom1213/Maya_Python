# A00240_PathTool — 사용 안내

자주 쓰는 **폴더 경로를 버튼으로 만들어 두고, 클릭하면 그 경로를 탐색기로 여는** 런처 툴.
버튼을 **카테고리**로 묶고, 다시 집/회사처럼 환경별 **프로파일**로 나눠 관리한다.
탭은 **ShortCut**(경로 버튼 런처)과 **Tree**(입력 경로를 트리뷰로 탐색, 10장)로 구성된다.

> 이 툴은 Maya 안에서 도는 툴이 아니라 `A00210_FileManager`·`A00220_BackupTool` 처럼
> **Windows 에서 독립 실행되는 PySide6 앱** 이다. Maya 설치/실행 없이 동작한다.

---

## 1. 핵심 개념

| 개념 | 설명 |
|------|------|
| **Profile** | 집/회사처럼 **서로 다른 환경의 경로 묶음**. 프로파일 1개 = JSON 파일 1개로 완전히 분리 저장된다. 콤보로 전환. |
| **Category** | 한 프로파일 안에서 경로 버튼을 묶는 그룹. 각 카테고리는 별도 `QGroupBox` 로 표시된다. |
| **Path 버튼** | 특정 폴더 경로 1개. **클릭하면 그 경로를 탐색기로 연다**(폴더는 그 폴더를, 파일이면 폴더를 열고 파일 선택). |

계층: **Profile → Category → Path 버튼**.

설정은 git 으로 공유하지 않고 **툴 폴더 내부 `data/`** 에 저장된다(자세히는 8장):
`<A00240_PathTool>/data/profiles/<name>.json` + `data/active.json`.

---

## 2. 실행

- **개발 실행**: `python JUN_All/tools/A00240_PathTool/launch.py`
- **exe 빌드**: 툴 폴더의 `build_exe.bat`(PyInstaller, `launch.spec`) → `dist/A00240_PathTool.exe`
- 필요 패키지: `PySide6`, `pyinstaller`. 경로 열기/저장 로직은 **표준 라이브러리**(os/sys/subprocess/json)만 쓴다.

## 3. 화면 구성 (ShortCut 탭)

```
┌ Profile ──────────────────────────────────┐
│ [ Home ▼ ]   [New] [Rename] [Delete]       │
└────────────────────────────────────────────┘
┌ Create ───────────────────────────────────┐
│ [Category]  [Path]                         │
└────────────────────────────────────────────┘
┌ Textures ─────────────────────────────────┐   ← 카테고리(QGroupBox)
│ [ char_tex ]                               │   ← Path 버튼(클릭=탐색기 열기)
│ [ env_tex  ]                               │
└────────────────────────────────────────────┘
┌ Scenes ───────────────────────────────────┐
│ [ shot_010 ]                               │
└────────────────────────────────────────────┘
```

> 탭은 앞으로 늘어날 예정이라 `QTabWidget` 으로 구성돼 있다(현재 **ShortCut** 1개).

---

## 4. 경로 버튼 만들기

1. **Category** 버튼 → 카테고리 이름 입력 → 새 카테고리(QGroupBox) 생성.
2. **Path** 버튼 → **한 다이얼로그**에서 아래 3가지를 한 번에 입력:
   - **Add to category**: 어느 카테고리에 넣을지(콤보)
   - **Button name**: 버튼에 보일 이름
   - **Path**: 폴더 경로(직접 입력 또는 `Browse...`)
   - 이름/경로가 비었거나 같은 카테고리에 같은 이름이 있으면 안내 후 **입력값을 유지한 채** 다시 묻는다.
3. 만들어진 **Path 버튼을 클릭**하면 그 경로가 탐색기로 열린다.
   (경로가 사라졌으면 경고만 띄우고 아무것도 열지 않는다.)

---

## 5. 수정 / 삭제 / 순서 변경 — 우클릭 메뉴

항목이 늘어나도 화면을 깨끗하게 유지하려고 **우클릭 컨텍스트 메뉴**로 처리한다.

| 우클릭 대상 | 메뉴 |
|-------------|------|
| **카테고리 헤더(빈 곳)** | **Move Up** / **Move Down** / **Rename Category** / **Delete Category** |
| **Path 버튼** | **Move Up** / **Move Down** / **Rename** / **Change Path** / **Change Category** / **Delete** |

- 카테고리 삭제는 그 안의 버튼까지 지우므로 확인 다이얼로그를 1회 띄운다.
- **Move Up / Move Down**: 카테고리뿐 아니라 **카테고리 안의 Path 버튼**도 이 메뉴로 한 칸씩 올리고/내려
  **원하는 순서로 재정렬**할 수 있다(맨 위/맨 아래 항목은 해당 방향이 비활성). 카테고리는 카테고리끼리,
  버튼은 그 카테고리 안에서만 이동한다. 바뀐 순서는 프로파일 JSON 에 저장된다.
- **Change Category**: 버튼을 **다른 카테고리로 이동**한다. 현재 카테고리를 뺀 목록에서 대상을 고르면 그쪽
  끝으로 옮겨진다(대상에 같은 이름 버튼이 있으면 막는다). 다른 카테고리가 없으면 안내한다.
- Path 버튼의 **평소 클릭은 경로 열기**, **우클릭이 편집 메뉴**다.

## 6. 프로파일 (집/회사 분리)

상단 **Profile** 그룹에서 환경별 경로 묶음을 나눠 관리한다.

- **콤보 전환**: 고른 프로파일의 카테고리/버튼만 즉시 로드된다.
- **New**: 새 프로파일 생성(빈 상태) 후 그쪽으로 전환.
- **Rename**: 현재 프로파일 이름 변경(활성 상태 유지).
- **Delete**: 현재 프로파일 삭제 후 남은 프로파일로 전환. **최소 1개는 항상 남긴다.**
- 마지막으로 쓰던 프로파일은 기억되어 다음 실행 시 그대로 열린다.
- 프로파일 이름은 곧 파일명이라, 파일명 금지문자(`\ / : * ? " < > |`)는 `_` 로 자동 정리된다.

> 예) `Home` 프로파일엔 집 PC 의 경로, `Work` 프로파일엔 회사 경로를 따로 담아 두고 콤보로 전환.

---

## 7. 사용 순서 (요약)

1. (필요하면) **Profile → New** 로 `Home`/`Work` 등 프로파일을 만든다.
2. **Category** 로 분류를 만든다(예: `Textures`, `Scenes`).
3. **Path** 로 카테고리·이름·경로를 한 번에 입력해 버튼을 추가한다.
4. 버튼을 **클릭**해 경로를 연다. 수정/삭제는 **우클릭**.

---

## 10. Tree 탭 — 경로를 트리뷰로 보기

입력한 폴더 경로를 **트리뷰**로 펼쳐 본다(A00210 Path Structure 의 트리 표시와 같은 취지의
탐색용 뷰). 저장/동기화는 없고, 그때그때 경로를 훑어 보여준다.

```
┌ Path Tree ────────────────────────────────────────────┐
│ Path [................................]  [Browse]      │
│ Depth [3] [✓ Show files] [File Types ▾]  [Build][Expand]│
└────────────────────────────────────────────────────────┘
 📁 assets
   📁 textures
       📄 wood.png
   📄 readme.txt
 📁 scenes
```

- **Path / Build Tree**: 경로를 지정(Browse 또는 직접 입력)하고 **Build Tree**(또는 Enter)로
  트리를 그린다. Browse 로 폴더를 고르면 바로 빌드된다.
- **Depth**: 보여줄 깊이. `3` 이면 3단계까지, **`0`(= `All`)** 이면 전체. 값을 바꾸면 자동 재빌드.
- **Show files**: 끄면 **폴더만**, 켜면 파일까지. (끄면 File Types 는 비활성.)
- **File Types ▾**: 스캔에서 발견된 확장자 중 **표시할 것만 체크**(All 포함). 여러 개 연속 토글해도
  메뉴가 닫히지 않고, 버튼 라벨에 선택 요약이 보인다. (A00210 File Manager 와 동일 동작.)
- **폴더/파일 구분**: 폴더·파일에 각각 폴더/파일 **아이콘**이 붙어 한눈에 구분된다.
- **Expand**: 창이 작을 때 **큰 창**(리사이즈 가능)에서 같은 트리를 본다.
- **우클릭 → Reveal in File Explorer**: 트리 항목을 우클릭하면 그 경로를 **탐색기에서 연다**
  (폴더는 폴더 열기, 파일은 폴더를 열고 파일 선택). 메인 트리와 Expand 창 모두 지원.

> Show files / File Types 는 **재스캔 없이 즉시** 반영된다(Build 시 모든 파일을 캐시해 두고 거른다).
> Depth·Path 변경만 다시 스캔한다.

## 8. 저장 위치 (개발 참고)

설정은 **툴 폴더 내부 `data/`** 에 저장된다(`%USERPROFILE%` 아님).

```
A00240_PathTool/data/
├── profiles/
│   ├── Home.json      # {"categories": [{"name","buttons":[{"name","path"}]}]}
│   └── Work.json
└── active.json        # {"active": "Home"}
```

- **onefile exe 주의**: PyInstaller `--onefile` 로 빌드하면 코드가 임시폴더(`_MEIPASS`)에
  풀리므로 거기에 쓰면 종료 시 사라진다. 그래서 **frozen 일 때는 `__file__` 이 아니라
  exe 가 있는 폴더 옆 `data/`** 에 저장한다(영속 보장). 소스 실행이면 툴 루트의 `data/`.
- `data/` 는 사용자 데이터라 **`.gitignore`** 대상(`**/A00240_PathTool/data/`).
- 구버전 `~/.jun_pathtool/shortcuts.json` 이 있으면 첫 실행 때 `Default` 프로파일로
  **자동 마이그레이션**(원본은 `.bak` 으로 남김).

## 9. 구조 (개발 참고)

```
A00240_PathTool/
├── launch.py                 # standalone 진입점 (green_dark 테마, 파이프라인/유틸 카테고리)
└── app/
    ├── config/version.py     # VERSION / LAST_UPDATE
    ├── core/
    │   ├── prefs.py          # 프로파일 영속화(목록/로드/저장/이름변경/삭제/active)
    │   ├── path_opener.py    # 경로를 탐색기로 열기 (Win/mac/linux)
    │   └── tree_scanner.py   # 경로를 깊이 제한으로 훑어 트리 노드(dict) 생성 (Tree 탭)
    └── ui/
        ├── main_window.py    # QTabWidget (ShortCut + Tree)
        ├── shortcut_tab.py   # ShortCut 탭 본체 + AddPathDialog
        └── tree_tab.py       # Tree 탭 본체 (트리뷰 + 깊이/파일토글/확장자필터/Expand/Reveal)
```

- `app/core`(로직)와 `app/ui`(화면)를 분리한다(`prefs`/`path_opener` 는 표준 라이브러리만 사용).
- import 는 **툴 고유 경로**(`tools.A00240_PathTool.app...`)와 내부 **상대 import** 로 한다 —
  여러 standalone 툴을 한 인터프리터에서 동시에 띄울 때 최상위 `app` 패키지가 충돌하지 않도록.
